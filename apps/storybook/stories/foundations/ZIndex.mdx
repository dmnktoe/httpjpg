# Z-Index

Named stacking layers. Reach for the token name instead of a magic number so layering stays
predictable across components. Values are spaced in steps to leave room for future insertions
without renumbering the world.

## Usage

Token names map to Panda's `zIndex` scale, so `zIndex: "modal"` resolves to the configured value.

```tsx
import { Box } from "@httpjpg/ui";

<Box css={{ position: "fixed", zIndex: "header" }}>Site header</Box>;
<Box css={{ position: "fixed", zIndex: "cookieBanner" }}>Cookie banner</Box>;
```

## Guidelines

- **Always use a named layer.** Never hard-code a raw `z-index` number in a component.
- **`hide` (-1)** pushes decorative elements behind their container; **`base` (0)** is the default flow.
- Some layers intentionally share a value (e.g. `modal` and `banner`) — order them with DOM order, not by nudging the number.
- Need a new layer? Add it to `@httpjpg/tokens` between two existing steps rather than inventing a local value.
